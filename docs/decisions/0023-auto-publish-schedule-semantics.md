# 0023. 자동 발행 스케줄 시맨틱 — 시작 시 백필은 예약 시각 경과 후에만

- 상태: accepted
- 작성일: 2026-06-15

## 맥락

데스크톱 자동 발행은 두 트리거를 공유한다: 앱 시작 시 `initAutoPublish()` 의 catch-up, 그리고 예약 시각(`cfg.time`) 타이머. 기존 `runAutoPublish` 의 `dueRuns` 는 시각·중복 체크 없이 `cfg.daily` 면 항상 daily 를 due 로 반환했다. 그 결과:

- **예약 시각과 무관하게 앱을 켜는 순간 오늘치 daily 가 발행됨.** (사용자 리포트: 알람 19:00 인데 18:21 에 앱을 열자 오늘 일지가 생성)
- 발행은 단일 메모리 락(`running`)을 ~2분 점유 → 그 사이 수동 발행이 충돌해 "이미 다른 작업이 실행 중입니다" raw 에러.

`runCore` 의 락·충돌 에러는 앱 내부 동시 실행 가드일 뿐, 외부 프로세스(예: CLI)와 무관하다.

## 결정

- 자동 발행 trigger 를 `'startup' | 'scheduled'` 로 구분한다.
- **startup catch-up 은 오늘 예약 시각이 이미 지난 경우에만** 실행한다(`isScheduledTimeReached(now, cfg.time)`, 로컬 TZ). 예약 시각 전이면 발행하지 않고 타이머에 위임 → 앱을 일찍 켜도 오늘치가 조기 발행되지 않는다.
- 락 충돌 throw 를 한국어 하드코딩 → `busy:<mode>` **코드**로 바꿔 렌더러가 i18n 으로 친화 문구 매핑(영어 사용자에 한국어 새는 것 방지).
- `cairn:busy` 를 모든 창에 브로드캐스트 → 렌더러가 자동 발행 백필 포함 모든 실행을 인지하고 발행 버튼을 잠근다.

## 대안

- **중복 발행을 멱등성에만 의존**: 발행자는 이미 발행된 날을 short-circuit 하지만(크레딧 무소모), "오늘치가 아직 없는 시점에 앱을 처음 켜면 조기 발행" 문제는 못 막아 기각.
- **startup catch-up 제거**: 놓친 날(앱이 꺼져 있던 동안) 백필이 안 돼 기각. 시각 게이트만 추가.

## 결과

- 앱을 예약 전에 켜도 조기 발행 없음. 예약 후에 켜면 놓친 발행 catch-up.
- 타임존 규칙 준수(로컬 `getHours/getMinutes`, KST/UTC 단정 없음).
- 구현: PR #146. 헬퍼 `isScheduledTimeReached` 는 pure·export(테스트 가능). desktop 테스트 러너 부재로 별도 스펙은 미동반.
