# 0002. Portable 배포 (개인/회사 노트북 모두 동일하게 셋업)

- 상태: accepted
- 작성일: 2026-04-26

## 맥락

사용자는 회사 GitHub·Notion·Slack을 회사 메일로 **개인 노트북에서도** 접근 가능. 따라서:
- 개발 + 실행을 한 머신(개인 노트북)에서 모두 가능
- 필요시 회사 노트북에서도 같은 도구를 돌리고 싶음

옵션:
1. 개인 노트북 한 대에 한정 — 가장 단순하지만 "두 대 다 쓰고 싶을 때" 막힘
2. 회사 노트북에서만 실행 — 개인 노트북에서 dev/test가 어려움
3. 어느 머신에서나 동일하게 동작 (portable)

## 결정

**Portable 구조**. 코드와 설정을 분리해서 어느 macOS 머신에서나 같은 절차로 셋업·실행 가능.

- 코드는 git clone 하나로 끝
- 설정은 머신별 (`worklog.config.json`, `.env`, launchd plist 위치)
- 시크릿(GitHub PAT, Notion 토큰)은 **머신마다 별도 발급** + 그 머신에만 저장. 토큰 복사 금지(키 회전 시 추적 끊김 방지)
- 두 머신에서 동시에 launchd 활성화하지 않음 (멱등성 충돌 가능). 한 시점에 한 머신만
- `MACHINE_NAME` env로 페이지 메타데이터에 발행 머신 기록

## 대안

- **A. Docker 이미지 배포** — Portable의 한 형태이지만 macOS launchd와 결합이 어색하고, NestJS standalone CLI에 컨테이너는 과함
- **B. 클라우드 배포 (Railway/Fly)** — 항상 켜있는 환경 → 개인노트북 슬립 무관. 단점: 과금 발생 (무과금 정책 위반), 로컬 git에 접근 못 함

## 결과

- **장점**: 어느 머신이든 git clone + pnpm install + 본인 .env로 시작. 실행 머신 변경 자유.
- **트레이드오프**:
  - 시크릿이 머신별로 분산 → 토큰 회전 시 머신마다 갱신 필요 (반대로 한 머신 잃어도 다른 머신은 영향 없음)
  - launchd 활성화는 한 머신만 — 사용자가 직접 관리해야 함
- **검증**: `docs/SETUP.md` (단계 8에서 작성)에 머신별 셋업 절차 명시
