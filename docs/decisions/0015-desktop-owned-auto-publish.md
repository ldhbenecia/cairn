# 0015. 데스크톱 앱이 자동 발행을 소유 + 크레딧 동의 정책

- 상태: accepted
- 작성일: 2026-06-03

## 맥락

자동 발행은 그동안 **launchd** 가 헤드리스 **core 엔진**을 정해진 시각 + 로그인 시(RunAtLoad)에 실행하는 구조였다(ops/install.sh). 데스크톱 앱은 UI/수동 발행만 담당했다.

일상 사용에서 드러난 문제:

- 앱을 켜도 밀린 날짜가 자동 처리되지 않는다(앱은 자동 발행과 연결돼 있지 않음). 사용자는 "껐다 켜면 처리되겠지" 라고 기대하지만 아무 일도 안 일어난다.
- 자동 발행이 도는지/언제 도는지 사용자가 알 방법이 없다.
- 자동 발행은 **사용자 본인의 Claude 크레딧**을 소비하는데, 동의를 받는 흐름이 없다.

데스크톱 앱은 배포 빌드에서 트레이 상주(ADR: dev 비상주/배포 상주)라, 자동 발행을 직접 소유할 수 있는 위치에 있다.

## 결정

**자동 발행 소유권을 데스크톱 앱으로 옮긴다.**

- 앱이 (a) 실행 시 밀린 날짜 백필, (b) 매일 지정 시각 발화, (c) 진행/결과 시스템 알림, (d) 동의를 직접 관리한다.
- **크레딧 동의 = opt-in**: 자동 발행은 기본 OFF. Preferences 에서 켜는 행위가 동의다. 추가로 "발행 전 확인" 옵션을 두어, 매 실행 전 알림으로 확인받고 진행할 수 있다.
- **알림에 cairn 로고**를 심어 출처를 분명히 한다.
- **launchd 는 제거하지 않는다** — 앱이 아예 떠 있지 않은 헤드리스/CLI 사용자를 위한 보조로 남긴다. 데스크톱 앱과 launchd 가 모두 트리거해도 publisher 의 already-published skip 으로 **중복 발행은 무해**하다.

## 대안

- **launchd 유지 + Schedule UI 로 plist 편집**: 앱이 plist XML 을 재작성하고 `launchctl bootout/bootstrap`. 패키징 앱은 엔진 경로가 앱 번들 안이라 plist 의 repo 경로 가정과 엇갈리고, 동의/가시성을 앱이 통제하기 어렵다. 기각.
- **자동 발행 기본 ON**: 크레딧을 동의 없이 소비 → 기각. opt-in 이 원칙.

## 결과

- settings 에 `autoPublish { enabled, time, backfillDays, confirmBeforeRun }` 추가.
- main 에 스케줄러(앱 ready/설정 변경 시 재설정, 실행 시 백필 + 매일 발화).
- Preferences 에 자동 발행 섹션 + 크레딧 안내.
- launchd 와 공존(중복은 skip 으로 무해). 추후 launchd 정리/이관은 별도 결정.
- weekly/monthly 자동 발화, confirmBeforeRun 의 인앱 확인 고도화는 후속.
