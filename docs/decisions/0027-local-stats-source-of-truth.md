# 0027. 대시보드 통계의 진실 소스를 로컬 파일로 (노션은 출력 전용)

- 상태: accepted
- 작성일: 2026-06-19

## 맥락

통계 대시보드(시간대별 차트 포함)와 롤업 metrics 가 노션 `Source counts` 속성(`gh:N / git:M`,
한때 `hrs:...` 히스토그램까지 포함)을 역으로 파싱해서 만들어졌다. 문제:

- **조작·삭제 가능**: 노션 속성은 사용자가 노션 UI 에서 직접 지우거나 고칠 수 있다. 그러면 통계가
  바뀌거나 사라진다. 실제로 `Source counts` 는 지저분해서 사용자가 지우고 싶어 했다.
- **노션 노출 부적절**: "몇 시에 몇 개 커밋" 같은 활동 시간대(`hrs:...`)를 노션 페이지 속성에
  노출할 이유가 없다. 가독성도 나빴다.
- cairn 철학상 **노션은 출력(읽을 일지) 대상이고 진실 소스는 로컬 엔진**이다(ADR 0002). 통계를
  출력물에서 역산하는 구조 자체가 뒤집혀 있었다.

이전에 `hrs` 를 별도 'Activity hours' 노션 속성으로 분리하는 안을 잠깐 구현했으나, 결국 노션에
노출되는 건 똑같아 폐기했다.

## 결정

발행된 일지의 통계(`pr`, `commit`, 시간대 `hours[24]`)를 **로컬 파일**
`~/.cairn/worklog-stats.json` 에 기록하고, 이를 진실 소스로 삼는다. key 는 `${category}:${date}`.

- **core**: 일간 발행 성공(`created`/`recreated`) 시 `WorklogStatsService.record('daily', date, {pr, commit, hours})`.
  `commit` = 그날 총 커밋(GitHub PR 커밋 + 로컬 커밋), `hours` = 커밋 시각 24칸 히스토그램(머신 로컬 TZ).
- **노션 `Source counts` 속성 제거**: 일간 페이지에 더는 쓰지 않고, 신규 DB 스키마에서도 뺀다.
  기존 DB 의 컬럼은 원격에서 지우지 않지만(사용자가 지워도 무관) 앱이 의존하지 않는다.
- **롤업 collector**: 일간 metrics 합산을 노션 `Source counts` 대신 로컬 stats 에서 읽는다.
- **데스크톱**: 대시보드·워크로그 리스트의 카운트/시간대를 로컬 stats 에서 읽는다(`RecentPage` 가
  `pr/commit/hours` 를 직접 보유, 노션 속성 파싱 제거). 노션은 제목·상태·링크·본문만.

## 대안

- **노션에 그대로 두기**: 조작·삭제·노출 문제 그대로. 기각.
- **별도 'Activity hours' 노션 속성**: 노션 노출은 여전. 기각(잠깐 구현 후 되돌림).
- **레거시 폴백(기존 페이지의 노션 `Source counts` 읽기)**: 사용자가 주간을 재발행해 레거시
  페이지를 없앨 예정이라 불필요. 넣지 않음.

## 결과

- 통계가 노션 편집과 무관하게 안정·불변. 노션 페이지가 깔끔해짐(`hrs` 도착 라인 제거).
- 단일 머신 가정(ADR 0002)에선 문제 없음. 다른 머신에서 보면 로컬 stats 가 없어 카운트가 빈다.
- 이 변경 **이전에 발행된 일간 페이지**는 로컬 stats 가 없어 대시보드/배지에서 카운트가 빈다.
  재발행하면 채워진다(폴백 없음).
- 롤업 `notionPageCount` 는 항상 0(노션 수집은 2026-06-03 제거됨) — 표기만 유지.
