# 0022. 요약 모델 선택 — 기본 Sonnet, 설정으로 변경

- 상태: accepted
- 작성일: 2026-06-14

## 맥락

발행 시간을 계측한 결과 collect 2.6s / summarize 123s / publish 2.1s 로, **요약이 전체의 96%** 였다 (사용자 최초 불만: "시간관련된게 처리속도가 굉장히 느리다"). 수집·발행은 이미 캐시·병렬화로 최적화돼 더 줄일 여지가 적고, 유일하게 큰 레버는 **요약 모델** 이다.

그동안 `query()` 에 `model` 을 지정하지 않아 Claude 로그인의 CLI 기본 모델(Max 플랜은 보통 Opus)로 요약했다. 그런데 요약 작업은:

- MCP 도구(`get_activity`/`submit_summary`)가 **구조화된 활동을 그대로 제공** 하고,
- 모델은 그걸 한국어로 수치 보존하며 정리만 하면 됨 (창의적 추론 부담 낮음).

이 작업은 Opus 가 과하다. Sonnet 으로도 품질이 충분하면서 훨씬 빠르다.

## 결정

요약에 쓸 모델을 **선택 가능** 하게 하고, **기본값을 Sonnet** 으로 둔다.

- core: `CAIRN_SUMMARY_MODEL` env (`sonnet`|`haiku`|`opus`) 를 읽어 daily·rollup 의 `query()` `options.model` 로 전달. `default`/미지정/알 수 없는 값은 override 하지 않음(CLI 기본 모델). 별칭은 Claude Code CLI 가 해석하므로 날짜 박힌 모델 ID 를 하드코딩하지 않는다.
- desktop: `settings.summaryModel`(기본 `sonnet`) + Preferences 프롬프트 탭의 세그먼트 선택 UI. `default` 가 아니면 core 에 env 로 전달.
- 모델은 **사용자 본인 Claude 로그인** 으로 돈다 → 모델을 바꿔도 추가 과금 없음(ADR 0001).
- egress 에는 영향 없음 — 보내는 payload 는 모델과 무관하게 동일하게 검사된다(ADR 0021).

## 대안

- **Opus 기본 유지**: 가장 느림. 사용자 최초 불만의 직접 원인. 기각(단, 옵션으로는 제공 — 롤업 최고 품질용).
- **Haiku 기본**: 가장 빠르나 한국어 수치화 요약·도구 사용에서 Sonnet 보다 품질 여유가 적음. 기본으론 과감하다고 봐 옵션으로만.
- **모델 ID 하드코딩**: `claude-sonnet-4-6` 등은 시간이 지나면 rot. 별칭(`sonnet`) 전달로 회피.

## 결과

- 기존 사용자도 다음 실행부터 기본 Sonnet 적용 → 체감 발행 속도 개선. 원하면 한 번의 클릭으로 Opus(최고 품질)·Haiku(가장 빠름)·기본(로그인 모델)으로 변경.
- 트레이드오프: 기본 모델을 사용자 로그인 기본값에서 Sonnet 으로 바꾸는 **동작 변경**. 명시적 설정 UI 와 ADR 로 가시화해 되돌리기 쉽게 함.
- 후속: 발행 결과 화면에 사용한 모델·소요 시간을 노출하면 사용자가 속도/품질 트레이드오프를 체감하기 좋음(백로그).
