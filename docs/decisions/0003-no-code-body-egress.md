# 0003. 코드 본문 / diff 외부 송신 절대 금지

- 상태: accepted
- 작성일: 2026-04-26

## 맥락

cairn은 회사 private GitHub 레포의 PR/커밋 데이터를 수집한다. 그중 일부를 Claude(외부 LLM API)로 보내 한국어 요약을 생성한다.

회사 코드는 영업 비밀·지식재산이며, 외부 API로 노출되면 보안·법적 리스크. 사용자는 단계 0에서 "private 레포지토리"를 명시했다 (= 외부 노출에 민감).

## 결정

회사 코드의 본문, diff, 파일 내용, 절대경로, 이메일 등은 **외부 API로 절대 송신하지 않는다.** 한 바이트도.

### 외부로 나가도 되는 것 (화이트리스트)
- PR 제목, 라벨, 머지 상태
- 변경 파일의 **이름만** (전체 경로/절대경로 X)
- 커밋 메시지 첫 줄 (제목)
- 커밋 short SHA (전체 X)
- Notion 페이지 제목, URL, 마지막 편집 시각
- repo basename

### 절대 외부로 나가면 안 되는 것
- diff, patch, hunk
- 파일 본문, 코드 스니펫
- 절대경로 (`/Users/...`)
- 이메일 주소, 토큰, API 키
- 사용자별 식별 정보

## 강제 수단

1. **타입 정의에서 제외**: 외부 송신 페이로드 타입에 코드/diff 필드 자체를 두지 않음. agent harness에 노출되는 도구 응답 타입도 같음.
2. **단위 테스트 스냅샷**: 송신 페이로드를 `JSON.stringify`해서 정규식 `diff|patch|@@|^---|^\+\+\+` 매칭이 없는지 검증. 매칭되면 테스트 실패.
3. **redaction 헬퍼**: 커밋 메시지에 `(api[_-]?key|token|secret|password)\s*[:=]\s*\S+` 패턴이 발견되면 마스킹.
4. **로그 redaction**: pino redact paths에 `*.token`, `*.api_key`, `headers.authorization`, `env.*PAT*` 등록.

## 대안

- **A. diff도 보내고 더 풍부한 요약** — 한국어 요약 품질 ↑. 단 보안 리스크 + 사용자가 명시적으로 거부.
- **B. LLM 자체를 안 씀** — 100% 안전하지만 한국어 단락 요약 품질 0.

## 결과

- **장점**: 정책 위반 가능성 자체를 차단 (테스트 실패하면 머지 불가)
- **트레이드오프**: 요약 품질이 메타데이터 수준에 의존. PR 제목/라벨/파일명이 잘 정리된 팀일수록 요약 품질 ↑
