## 완료

- **설정 저장소** `~/.cairn/settings.json` (main 소유) — `src/main/settings.ts` readSettings/writeSettings. → ADR 0013
  - 무플래시: preload `sendSync('cairn:bootstrap-sync')` → `{ settings, version }` 동기 수신, `window.cairn.initialSettings` 노출
  - `cairn:settings:set` IPC, version 하드코딩 제거(app.getVersion)
- **테마** 다크/라이트/시스템 — styles.css `[data-theme=light]` 토큰 override, `applyTheme()` (main.tsx 무플래시 + SettingsProvider 동기화, system 은 matchMedia 추종)
- **i18n** ko/en — `i18n.ts` 경량 커스텀(의존성 X). → ADR 0014
  - `SettingsProvider` 가 `t()` 제공. sidebar / worklog-list / publish-dialog / preferences 문자열 전환
- **알림 on/off** — notifier 가 `readSettings().notifications` 존중
- **Preferences 개편** — config 덤프 → 섹션(Appearance 테마·Language·Notifications·Config·About) + 설정 연동

## 시행착오 / 결정

- preload `sandbox: true` → node fs 불가 → 무플래시는 `ipcRenderer.sendSync` 로 main 동기 수신 (ADR 0013)
- 테마 토큰: Tailwind v4 `@theme` 를 다크 기본(:root)으로 두고 `[data-theme=light]` 에서 같은 `--color-*` override → 유틸 클래스 자동 반영
- i18n: react-i18next 대신 경량 dict (2개 언어·정적) (ADR 0014)
- 카테고리/status 칩 색은 다크 기준 하드코딩 — 라이트에서 대비 약할 수 있음(후속 polish)
