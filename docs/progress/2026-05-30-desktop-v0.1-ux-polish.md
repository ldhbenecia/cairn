## 시행착오 / 결정

- **step indicator 패턴 = stdout line matching** — engine 변경 X. core-runner 가 stdout 의 핵심 line 패턴 (예: `github collect done` / `summarizer finished` / `worklog page created` / `orchestrator.run done`) 매치 → step transition 감지 → renderer 에 IPC `cairn:run-step` emit. raw log tail 은 "자세히 보기" 토글로 숨김 처리 (production v1 UX 의 1차 형태)
- **Recent panel = 노션 API 호출 (main process)** — `NotionCollectorService` 의 worklog DB 조회 패턴 재사용 가능. 다만 core 의 collector 를 직접 import 하면 무거우니 desktop 의 별도 가벼운 client 또는 core spawn 시 한 번에 묶기. 14.6 의 첫 시도는 별도 main 모듈에서 노션 API 직접 호출
- **Logs panel** — `~/.cairn/logs/cairn-*.log` 최신 파일의 tail 마지막 200 줄. 실시간 watch 는 14.6 에서 안 함 (v0.5+ 의 별도 영역)
- **Settings panel** — `worklog.config.json` 의 내용을 JSON tree 또는 dl/dt 으로 표시. 편집은 v0.2 셋업 마법사
- **animation = CSS @keyframes 직접** — Tailwind v4 의 `animate-in` 은 plugin 의존이라 styles.css 에 `panel-enter` 키프레임 (180ms opacity + translateY)
- **worklog.config.example.json 갱신** — schema 의 worklog.databaseId / dataSourceId / rollup 까지 반영. 이전 example 이 pageId 만 있어 schema 와 불일치했던 거 fix
- **v0.2 config 위치 이전 메모** — 사용자 지적: 현재 cairn root 의존 = personal v0.1 한정. v0.2 셋업 마법사 시점에 `~/.cairn/worklog.config.json` 으로 이전 + GUI 편집 (코드 안 까도 되도록). v0.3 OAuth Device Flow + v0.4 Keychain 으로 토큰 셋업도 GUI 흡수. Settings panel 의 `readConfig()` path 한 줄 교체만으로 호환 — 격리 좋음. 별도 ADR 후보 ("0014-config-location-home-dir")
- **Recent panel = v0.5+ placeholder** — 노션 DB 조회 + 인앱 viewer + 양방향 편집은 v0.5+ 단위. 14.6 에선 "v0.5+ 에 등장" 안내만
