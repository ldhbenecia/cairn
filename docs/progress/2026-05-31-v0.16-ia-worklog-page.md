## 시행착오 / 결정

- Recent 위치: 우측 rail(Linear) vs 하단 통합 → **하단 통합** 채택. 기능 수 대비 우측 rail은 과함
- Recent 는 mode 무관하게 페이지 하단에 항상. mode tabs 는 발행 액션용, 표는 카테고리 섞인 단일 목록. RunPanel 내부 RecentMini + weekly/monthly placeholder 는 페이지 레벨 Recent 로 흡수
- **작성자(아바타) 표시 불가**: worklog/rollup DB 에 People property 없음(전부 integration 생성, 개인 워크로그라 작성자 항상 본인). → workspace 칩으로 대체. 필요 시 myUserId 로 본인 아바타 가져올 순 있으나 redundant
- create 코드 스키마 ≠ 라이브 스키마. 코드엔 Title/Date/Status 만, 라이브엔 Tags/Source counts/Created·Last edited 추가됨. **실측 후 반영** (probe script)
