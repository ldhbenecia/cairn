# 2026-06-15 — publish-custom-date

> 진행 단계: 발행 다이얼로그 — 임의 날짜/주/달 (사용자 피드백)
> 상태: 완료

## 완료
- **발행 다이얼로그 날짜 picker** — 오늘/이번주/이번달만 되던 것을 임의 날짜로 확장. 모드 라벨을 일간/주간/월간으로, 날짜 입력(`max=today`) 추가. daily=그 날, weekly/monthly=그 날짜가 속한 주/달.
- core 의 `--date` 를 재사용(`CoreRunOptions.date`). 새 core 경로 없이 UI 만으로 구현.

## 시행착오 / 결정
- **`date` 는 오늘이 아닐 때만 전달**: 항상 보내면 `dateExplicit` 가 되어 백필이 단일 날짜 경로로 빠진다. 오늘이면 생략 → 기존 기본 동작(daily=오늘·weekly=지난주·monthly=지난달·백필) 보존. 과거 날짜만 명시.
- 과거 날짜 선택 시 daily 백필 토글은 비활성(백필은 "오늘 기준 최근 N일" 의미).
- 완전 임의 범위(시작~끝)는 새 core 모드가 필요 — 이번 범위 밖(사용자 "특정 날짜/주/달" 선택).
