## 목표

`worklog.config.json` + `.env` 를 손으로 작성해야 하는 진입장벽 제거. 앱 안에서 토큰 발급 링크 + 붙여넣기 + 연결 테스트 → 파일 자동 작성.

## 코어 사실 (조사 결과)

- **config 필수 필드** (notionWorkspaces[]): `label`, `tokenEnv`, `myUserId`(uuid 필수), `worklog.pageId`. `databaseId`/`dataSourceId` 는 **첫 발행 시 pageId 아래 자동 생성**(ensureDatabaseAndDataSource) → 위저드는 **pageId 만** 받으면 됨. rollup 도 동일
- **githubAccounts[]**: `label`, `tokenEnv`
- **.env 키**: `NOTION_TOKEN_<label>`, `GITHUB_TOKEN_<label>`, (선택) `ANTHROPIC_API_KEY`
- **Anthropic 인증** (ADR 0009 멀티오토): ① `ANTHROPIC_API_KEY` 있으면 사용 ② 없으면 **Claude Code CLI 로컬 인증 자동 인계(~/.claude, 추가 과금 0)** ③ 둘 다 실패 시 요약 skip + stub. → 온보딩은 **토큰 강제 X**, "claude 로그인돼 있으면 자동, 아니면 API key(선택)" 안내 + 테스트
- **myUserId**: 노션 integration 토큰으로 `users.list()` → `type==='person'` 인 사용자. 1명이면 그걸로, 여러 명이면 선택. (개인 워크스페이스만 trace 정책이라 보통 1명)

## 위저드 단계 (v1 = 단일 노션 워크스페이스 + 단일 GitHub 계정)

1. **Welcome** — cairn 소개 + 필요한 것 3가지(노션/깃허브/Claude) 안내
2. **Notion**
   - "integration 만들기" 링크(notion.so/my-integrations) + 토큰 붙여넣기(마스킹)
   - "부모 페이지" — 노션 페이지 **URL 붙여넣기** → pageId 추출(32hex→uuid). 그 페이지에 integration 을 connect 했는지 안내
   - **테스트**: 토큰으로 `users.list`(→ myUserId 후보) + 페이지 접근 확인. 실패 시 사유 표시
3. **GitHub**
   - "PAT 만들기" 링크(github.com/settings/tokens) + 필요 scope 안내(`repo`, `read:user`) + 붙여넣기
   - 라벨(기본 personal) + **테스트**: `GET /user`
4. **Claude(Anthropic)**
   - 설명: "요약에 Claude 사용. Claude Code CLI 가 로그인돼 있으면 **자동(추가 과금 0)**." + (선택) API key 붙여넣기(console.anthropic.com)
   - **테스트(선택)**: 작은 요약 호출로 인증 확인 — quota 소모 가능하니 버튼形 optional
5. **로컬 Git (선택)** — repo 경로 추가(폴더 선택 다이얼로그)
6. **완료** — `~/.cairn/.env` + `worklog.config.json` 작성(main) → 메인 진입

## 파일 작성 (main 책임)

- `.env`: 기존 보존 + 키 추가/갱신 (NOTION_TOKEN_<label>, GITHUB_TOKEN_<label>, ANTHROPIC_API_KEY?)
- `worklog.config.json`:
  ```jsonc
  { "localGitRepos": [...],
    "githubAccounts": [{ "label": "personal", "tokenEnv": "GITHUB_TOKEN_PERSONAL" }],
    "notionWorkspaces": [{ "label": "personal", "tokenEnv": "NOTION_TOKEN_PERSONAL",
      "myUserId": "<uuid>", "worklog": { "pageId": "<uuid>" } }] }
  ```
- 토큰은 `.env`(머신 로컬, ADR 0002). 입력 필드 마스킹, 로그 redact

## IPC (신규)

- `onboarding:test-notion(token)` → `{ ok, persons: [{id,name}], pageOk }`
- `onboarding:test-github(token)` → `{ ok, login }`
- `onboarding:test-anthropic()` → `{ ok }` (선택)
- `onboarding:finish(payload)` → .env + config 작성, `{ ok }`
- main 이 노션/깃허브 호출 (렌더러에서 직접 X — 토큰이 렌더러 거치지만 IPC 로 main 이 실행). 토큰은 작성 시점까지 메모리

## 확정된 결정 (2026-05-31)

1. **노션 부모 페이지** — **노션 검색 API 로 목록 선택**. integration 접근 가능한 page 들을 `search`(filter object=page)로 받아 제목+아이콘 목록 → 선택. (URL 붙여넣기 대신)
2. **연결 테스트** — 노션·GitHub 는 **입력 즉시 라이브 테스트**. Claude 는 quota 소모라 **선택 버튼**
3. **범위** — **멀티 계정 지원**. GitHub 여러 계정 추가/삭제, 노션도 복수 워크스페이스 추가 가능(각각 토큰+페이지). 라벨 사용자 지정
4. myUserId — `users.list` 의 person 1명이면 자동, 복수면 선택 UI

## 멀티 계정 UX (결정 3 반영)

- GitHub: "계정 추가" 반복 — 각 { 라벨, PAT, 테스트 }. 최소 1개
- 노션: "워크스페이스 추가" 반복 — 각 { 라벨, 토큰, 검색→페이지 선택, myUserId, 테스트 }. 최소 1개
- env 키는 라벨 기반: `GITHUB_TOKEN_<LABEL>`, `NOTION_TOKEN_<LABEL>` (라벨 대문자·정규화)
