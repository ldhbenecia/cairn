### 표 뷰 패턴 (사용자 추가 reference)

사용자 의도 — Linear 의 issue list / Pulse feed 같은 **dense 한 표 뷰**:
- 한 줄 = 한 entity (issue / worklog)
- columns: status badge (color-coded) + ID/date prefix + title + meta (assignee / workspace / time)
- 정렬 가능 (날짜 desc default)
- group by (date / status / workspace)
- compact padding (py-1.5 정도)
- hover row 강조 + 클릭 selection
- 우측 secondary panel 가능 (선택된 row 의 detail)
- 키보드 navigation (j/k 또는 ↑↓)

cairn 적용:
- Worklog 통합 page = 한 표 (mode tab 또는 group: Today / This week / This month / Past)
- columns: date · status badge (draft/final/skipped) · title · workspace · 마지막 수정 · 액션
- 클릭 시 우측 detail panel (또는 v0.5+ 인앱 viewer)
- 페이지네이션 또는 infinite scroll
- 검색 input + filter dropdown (workspace / status / mode)

### 16.1 — Information Architecture 재편 (사용자 지적 핵심)

**현재 구조의 문제**:
- sidebar 의 nav 항목 (오늘 / 이번 주 / 이번 달 / 최근 노션 / 로그 / 설정) 이 너무 분리
- 한 mode 당 빈 panel — 공간 활용 별로
- 최근 노션 페이지 = 별도 항목 → 통합 필요
- 로그 = production 에선 X

**v0.16 의 새 IA (Linear 패턴 참고)**:

```
[Sidebar]              [Main]
─────────────         ──────────────────────────────────
Worklog (default)      Header: Worklog
  ├─ Today               [mode tabs: Today | Week | Month]
  ├─ This week           [action group: 발행 버튼 + 옵션 토글]
  └─ This month          [status / step indicator / result card]
                         ─────────────
                         Recent (right rail 또는 하단 통합)
                         [최근 발행 페이지 list, 클릭 시 inline detail 또는 외부]

Preferences            Header: Preferences
                         테마 / Prompt / Notion / Workspaces / About

(로그 = 제거, 또는 Preferences 안 "Advanced > Debug logs" 토글)
```

핵심:
- **단일 "Worklog" 페이지** + mode tabs (Today / Week / Month). sidebar 의 3 개 → 1 개
- **Recent = 통합** — Worklog 페이지의 right rail 또는 하단. 별도 sidebar 항목 X
- **로그 = 제거** (Preferences > Advanced 안 토글)
- **Preferences = 통합 settings** (현재 "설정" 의 확장)

### 16.2 — UI 본격 polish (sidebar / Worklog / Recent)

- sidebar nav hover / active 의 좌측 indicator bar (Linear 패턴) — 4px width 의 active indicator
- sidebar footer = status bar:
  - 마지막 실행 시각
  - 다음 자동 실행 (launchd 의 schedule)
  - workspace 별 token status (online dot · green / red)
  - cairn 버전
- Worklog page = action area + step indicator + result card 의 spacing / typography 정돈
- Recent table → **페이지네이션** (또는 infinite scroll. 10/20/50 per page)
- Recent 안 **검색 input** (제목 / 날짜 fuzzy match)
- 트레이 brand 아이콘 PNG (현재 텍스트 'cairn')
- empty state — 발행 0 일 때 / token 없을 때의 안내 + setup CTA
- workspace status indicator — Recent 의 row 옆에 workspace dot color

### 16.3 — Preferences panel

- Sidebar 의 "설정" → "Preferences" 로 rename + 본격 contents:
  - **Appearance** — 다크 / 라이트 / system 테마 토글 (16.4 와 같이)
  - **Notion** — workspace list (token health check / 부모 page 변경 / 재인증)
  - **Schedule** — launchd 의 발화 시각 (현재 매일 시각 변경 UI)
  - **Worklog** — backfill days default / force default
  - **Advanced** — debug log 표시 토글 / config.json 직접 보기
  - **About** — cairn 버전 / link to repo / 업데이트 확인

### 16.4 — 다크 / 라이트 모드 토글

- styles.css 의 `:root` (다크 default) + `[data-theme="light"]` 토큰 fork
- system 토글 = `@media (prefers-color-scheme: light)` 도 같이
- light mode 토큰 = storm slate 의 어두운 변종 (canvas #f7f8f8, accent #2a3a4c 등)
- 별도 ADR (`0015-light-mode-tokens` 후보)

### 16.5 — Command palette (⌘K) + 키보드 단축키

Linear 의 핵심. cairn 도 동일 패턴 도입.

- ⌘K → palette 열림 → 검색 / 명령 (예: "오늘 발행", "최근 노션 페이지 N 열기", "Preferences 열기" 등)
- 광범위한 단축키:
  - ⌘1/2/3 = mode (이미 있음, tray)
  - `r` = 마지막 mode 재발행 (force)
  - `e` = 선택된 row 외부 열기
  - `n` = 새로고침
  - `/` = 검색 focus
  - `,` = Preferences
- Radix `Command` (or `cmdk` 라이브러리) 도입

### 16.6 — 셋업 마법사 + 노션 페이지 등록 튜토리얼

v0.2 셋업 마법사 영역과 합쳐서:
- 첫 실행 시 onboarding (welcome → notion integration token → 부모 page picker → github token → repo picker → 검증)
- 인앱 step indicator (튜토리얼)
- config 위치 = `~/.cairn/worklog.config.json` 이전 (ADR `0014-config-location-home-dir`)

## v0.17 — 사용자 prompt 커스텀

사용자 명시: "지금 작업일지 프롬포트 내가 정의해놓은거 쓰는데 이것도 사용자 커스텀 할 수 있게".

- cairn engine 의 hard-coded prompt (현재 `DailySummarizerService` / `RollupSummarizerService` 안에 있는 system prompt) → 사용자 정의 가능
- Preferences 안 "Summarizer" 섹션:
  - 모드별 (daily / weekly / monthly) prompt template 편집기 (multi-line text, monospace)
  - 변수 토큰 ($date, $activities, $sourceCounts 등) 정의
  - cairn engine 이 worklog.config.json 또는 별도 `~/.cairn/prompts/` 에서 사용자 prompt 우선 load + fallback 기본
  - 토큰 사용량 추정 미리보기 (선택)
- 별도 ADR (`0016-user-defined-prompts` 후보)
- engine 변경 분량 큼 (PR 시리즈 17.1 ~ 17.x)
