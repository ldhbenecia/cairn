## 단계 14 의 목적

두 가지 동시 달성:
1. **알림 클릭 사이드이펙트 해결** — 현재 engine 이 `osascript -e 'display notification'` 으로 알림. macOS 가 owner 를 Script Editor 로 인식 → 알림 클릭하면 Script Editor + iCloud 스크립트 picker 가 뜸. cairn 전용 bundle ID 가진 `.app` 이 native Notification 으로 보내야 진짜 해결.
2. **수동 trigger / 상태 / 로그 / 최근 노션 페이지 GUI** — engine CLI 직접 실행 / log tail / 노션 마지막 페이지 URL 기억의 GUI 흡수.

v0.1 = "shell". 셋업 마법사 / OAuth / Keychain 은 v0.2 이후. launchd 자동 발화는 v0.1 에선 그대로 (engine 직접 실행 + osascript 알림 사이드이펙트 유지) — v0.5+ "launchd → desktop owner" 패턴 옮길 때 같이 해결.

## 범위 (in / out)

**in**:
- `packages/desktop/` Electron + Vite + React + TS 스캐폴드
- 메뉴바 트레이 아이콘 + 우클릭 메뉴
- 작은 윈도우 (status / 수동 trigger / 로그 viewer / 최근 노션 페이지)
- core spawn (`child_process.spawn(node, '../core/dist/main.js')`)
- native Notification (cairn 번들 ID) — 수동 trigger 결과 알림
- 디자인 토큰 (Linear/Framer/Apple 톤, 다크 기본)
- electron-builder 로컬 빌드 (`.app` 산출) — 코드 사이닝 / notarization X
- 본인 머신 설치 + 동작 확인

**out (v0.2+ 로 미룸)**:
- 셋업 마법사 (`worklog.config.json` GUI 편집)
- OAuth Device Flow (GitHub / Notion)
- Keychain 토큰 저장
- 자동 업데이트
- 코드 사이닝 + notarization
- launchd 자동 발화의 owner 이전 (v0.5+)

| 영역 | 채택 | 이유 |
|------|------|------|
| 데스크탑 셸 | Electron | ADR 0010 |

| 번들러 | Vite | Electron + React + TS 표준 + HMR 빠름 |
| Electron 통합 | electron-vite | main / preload / renderer 동시 빌드, vite 표준 패턴 |

| 컴포넌트 primitive | Radix UI | 키보드 / 접근성 / 비스타일 primitive → Linear 톤 커스터마이즈 자유 |

| 아이콘 | lucide-react | 라인 아이콘, Linear 톤과 결 같음 |
| 빌드 / 패키징 | electron-builder | macOS `.app` / `.dmg` 산출 표준 |

### v0.1 이 React/Tailwind 인 이유 (vanilla 가 아닌 이유)

- Linear / Framer / Apple 톤은 디자인 토큰 (색·spacing·radius·shadow) 일관성이 핵심. vanilla CSS 로도 가능하지만 settings / log viewer / status 카드 들어가는 순간 토큰 관리가 무너짐.
- React + Tailwind v4 의 `@theme` directive 가 토큰을 CSS variable 로 1:1 매핑 → 다크/라이트 토글 / brand color 변경 0 비용.
- Radix primitive 는 Tooltip / Dialog / DropdownMenu / Tabs 같은 트레이 앱이 결국 다 필요한 요소들. 직접 키보드 / 포커스 / 접근성 짜는 비용 회피.
- 셋업 마법사 (v0.2) 진입 시 폼 / 검증 / 다단계 UI 가 들어오므로 어차피 React 필요 — 한 번에 가는 게 합리적.

## 디자인 시스템 (Linear / Framer / Apple 톤)

### 톤 가이드

- **다크 기본** — 작은 트레이 윈도우는 어두운 배경이 시각 부담 적고 Linear/Framer 류와 결 같음
- **좁고 정밀한 타이포** — Inter UI sans + JetBrains Mono (로그)
- **작은 radius** — 6~8px (Apple Settings 류). Tailwind `rounded-md` 기본
- **subtle 보더** — 1px, 낮은 대비 (`border-white/8` 류)
- **accent 1 색** — cairn 자체 컬러 (정해야 함, 잠정 cairn 의 돌탑 컨셉 → 따뜻한 stone/amber 계열 또는 Linear 의 indigo). v0.1 진입 시 정함
- **shadow 최소** — 트레이 윈도우 자체가 작아 elevation 표현 거의 안 씀
- **공간** — 4 / 8 / 12 / 16 / 24 px 스케일 (Tailwind 기본)
- **키보드 우선** — 모든 트레이 메뉴 / 윈도우 액션 단축키 (cmd+r 새로고침 / cmd+w 닫기 / cmd+1/2/3 일/주/월 trigger 등). Radix DropdownMenu 가 화살표키 기본 처리

### 토큰 초안 (CSS variables, v0.1 진입 시 확정)

```css
@theme {
  --color-bg: #0a0a0a;           /* 배경 */
  --color-surface: #141414;      /* 카드 */
  --color-border: #ffffff14;     /* 8% white */
  --color-text: #ededed;
  --color-text-muted: #8a8a8a;
  --color-accent: #8b7355;       /* 잠정 stone/amber — 정해야 함 */
  --radius: 6px;
  --font-sans: 'Inter', system-ui;
  --font-mono: 'JetBrains Mono', 'SF Mono', monospace;
}
```

→ Linear 실제 값은 v0.1 진입 시 본인 머신 브라우저 DevTools 로 inspect 해서 보정. `getdesign` CLI (`npx getdesign add linear.app`) 도 시도해서 DESIGN.md 받을지 결정.

## 핵심 동작 흐름

### 1. core spawn

- `core-runner.ts` 가 `child_process.spawn(process.execPath, ['../core/dist/main.js', '--daily'])` 같이 실행. Electron 의 `process.execPath` 가 번들된 node 라 별도 node 설치 불요.
- 모드 인자: `--daily` / `--weekly` / `--monthly` / `--backfill <days>` (core 가 이미 지원 — v0.11 backfill 포함)
- stdout / stderr 를 renderer 로 IPC 스트림 (log viewer 가 실시간 표시)
- exit code 0 → 성공, non-0 → 실패. notifier 가 결과 알림 발사.

### 2. native Notification (알림 사이드이펙트 해결)

- `new Notification({ title, body }).on('click', () => shell.openExternal(notionUrl))`
- 발사 owner = cairn 번들 ID → 클릭 시 Script Editor 안 뜸
- 클릭 시 발행된 노션 페이지 URL 열기 (state.ts 에 마지막 URL 기억)

### 3. 트레이 메뉴

```
cairn  ●           ← 상태 dot (idle / running / error)
─────────────────
오늘 일지 발행      ⌘1
이번 주 정리        ⌘2
이번 달 정리        ⌘3
─────────────────
지난 N 일 backfill ▶
─────────────────
대시보드 열기       ⌘D
로그 폴더 열기
최근 노션 페이지     ▶
─────────────────
Quit               ⌘Q
```

- "대시보드 열기" → 작은 윈도우 toggle
- "로그 폴더 열기" → `shell.openPath('~/.cairn/logs/')`
- "최근 노션 페이지" → 최근 5 개 sub-menu

### 4. 작은 윈도우

- 트레이 클릭 시 anchor 아래 토글
- 섹션: **상태** (마지막 실행 / 성공 / 실패), **수동 trigger** (3 버튼), **로그 tail** (실시간), **최근 노션 페이지** (링크 리스트)
- 폭 ~ 380px, 높이 ~ 540px. Linear 의 정보 밀도 참고

### 5. launchd 자동 발화 (v0.1 변경 X)

- 기존 ops/install.sh 가 등록한 launchd plist 가 그대로 engine 직접 실행
- → 자동 발화 알림은 여전히 osascript 사이드이펙트
- 수동 trigger 만 cairn 번들로 native Notification → 깔끔
- v0.5+ 에서 "launchd → desktop 앱 실행 → desktop 이 core spawn" 으로 옮기면서 전체 통일

## PR 시리즈 (의미 단위 커밋)

단계 14 는 한 PR 이 아니라 여러 PR 로 쪼갬. 한 PR = 한 의미 단위 + 직렬화.

**버전 정책 (dual-track)** — engine 과 desktop 은 개발 주기가 다르므로 **자체 lifecycle** 로 버전 관리. roadmap (`2026-05-17-cairn-v2-roadmap.md`) 의 "두 트랙은 독립적으로 릴리스" 따름.

- **root (`cairn-workspace`)**: progress 진행 단계 신호. ADR 0005 매핑 (PR patch, 단계 완료 minor) 유지.
- **`@cairn/core`**: engine 트랙 자체 lifecycle. 단계 14 는 core 변경 X → core 는 v0.14.0 유지.
- **`@cairn/desktop`**: desktop 트랙 자체 lifecycle. v0.0.x 부터 시작, v0.1 셸 (단계 14) 완료 시 v0.1.0.

이 dual-track 정책은 ADR 0005 단일 package 시절 매핑의 monorepo 확장 — 단계 14 끝나면 별도 ADR (0013 후보) 로 승격.

## 보안 / 정책 가드 (유지)

- ADR 0003 — desktop 이 코드 본문 / diff 외부 송신 X. core 가 sanitize 한 결과만 받음 (현재 core 가 LLM 으로 보내는 데이터 그대로 desktop 도 sanitize 통과한 것만 IPC 노출).
- IPC 채널 화이트리스트 — preload 에서 `contextBridge.exposeInMainWorld` 로 명시적 노출, `nodeIntegration: false`, `contextIsolation: true`, `sandbox: true`.
- 로그 viewer 가 보여주는 `~/.cairn/logs/` 는 이미 pino redaction 통과한 결과물. desktop 은 그대로 표시.
