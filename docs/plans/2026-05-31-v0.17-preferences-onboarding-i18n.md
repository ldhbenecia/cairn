## 공통 아키텍처 결정 (선결)

### 설정 저장소 — `~/.cairn/settings.json` (main 소유)

테마·언어는 렌더러 관심사, 알림은 main(notifier), 커스텀 프롬프트는 core(엔진) 관심사 → **3 프로세스가 공유**해야 함. 따라서 단일 파일:

- 위치: `~/.cairn/settings.json` (worklog.config.json 과 분리 — 그건 엔진 데이터 소스 config, 이건 사용자 환경설정)
- **main 이 소유** (read/write). 렌더러는 IPC(`settings:get` / `settings:set`)로 접근
- **무플래시 초기 로드**: preload 에서 시작 시 파일을 동기 읽어 `window.cairn.initialSettings` 로 노출 → 렌더러가 첫 페인트 전에 테마/언어 적용
- core 는 실행 시 이 파일을 읽어 customPrompt override 적용
- 스키마(초안):
  ```jsonc
  {
    "theme": "dark" | "light" | "system",   // 기본 dark
    "language": "ko" | "en",                 // 기본 ko
    "notifications": true,                    // 기본 true
    "prompts": { "daily": null, "weekly": null, "monthly": null } // null = 기본 프롬프트
  }
  ```
- → **ADR 후보**: 설정 저장소 위치/소유 (worklog.config.json 확장 X, 별도 settings.json)

### i18n — 경량 커스텀 (의존성 X)

- 앱이 작고 2개 언어(ko/en)뿐 → react-i18next 대신 **타입 안전 dict + `useT()` 훅 + Context**
- `src/renderer/src/i18n/{ko,en}.ts` (동일 키), `t('key')`, 언어는 settings 에서
- 기본 ko. 누락 키는 ko fallback
- → **ADR 후보**: i18n 방식 (경량 커스텀 채택 이유)

### 테마 — CSS 변수 + `data-theme`

- 현재 `@theme` 토큰(styles.css)을 `:root`(다크 기본) + `[data-theme="light"]` override 구조로
- 토글 시 `<html data-theme>` 설정. `system` 은 `prefers-color-scheme` 추종
- accent 인디고는 양 모드 공통(라이트에선 살짝 진한 변종 가능). surface/ink/hairline 라이트 팔레트 신규 정의
- 무플래시: preload initialSettings.theme 로 첫 페인트 전 `data-theme` 세팅

## PR-2 · 온보딩 / 셋업 위저드  *(가장 큼, 진입장벽 핵심)*

**문제**: 사용자가 `worklog.config.json` + `.env` 를 손으로 작성해야 함.

**범위**:
- **첫 실행/설정 누락 감지** → 온보딩 화면 (메인 대신 표시). empty state 의 CTA 와 연결
- 단계별 위저드:
  1. Welcome (cairn 소개)
  2. **Notion**: integration 생성 링크 + 토큰 붙여넣기 + 부모 페이지 선택(검색/URL 입력) + 연결 테스트
  3. **GitHub**: PAT 발급 링크(필요 scope 안내) + 붙여넣기 + 계정 라벨
  4. **Anthropic**: API key 발급 링크 + 붙여넣기
  5. (선택) 로컬 Git repo 경로 추가
  6. 연결 테스트(health check) → **`~/.cairn/.env` + `worklog.config.json` 자동 작성** (main 이 기록, 사용자는 JSON 안 만짐)
- 토큰은 `.env`(ADR 0002 머신 로컬 시크릿), 비밀번호 매니저 아님 — 평문 .env 유지 (현 정책). 입력 필드는 마스킹
- 재진입: Preferences 에서 "셋업 다시 실행" 가능

**리스크/미정**: Notion 부모 페이지 선택 UX(검색 API vs URL 붙여넣기), 토큰 유효성 체크 깊이. PR-2 착수 전 별도 세부 plan 권장.

**ADR**: 온보딩 토큰 입력/저장 흐름(NNNN)
