# 2026-04-27 — 데스크톱 뷰어 아이디어

> 상태: 아이디어 (실행 시점 미정)

## 떠오른 것

- 지금 cairn은 "수집 → 요약 → Notion 발행" 파이프라인. 산출물은 Notion에 쌓이지만, **모아 놓고 한눈에 훑는 뷰는 Notion UI에 종속**.
- 별도 데스크톱 앱으로 만들어서, Linear 같은 깔끔한 정보 밀도·타이포그래피·키보드 우선 인터랙션으로 **자기 일지/PR/커밋 타임라인을 정리해서 보는 뷰**가 있으면 좋을 듯.
- 잘 정리되어 있는 걸 보는 것 자체가 동기 부여 + 회고 도구가 됨.

## 확장 / 범용화 가능성

- 지금은 본인 1인용이지만, 같은 컨셉(GitHub + Notion + Git 활동 → 자동 일지)을 **다른 개발자들도 쓸 수 있는 범용 도구**로 키울 여지.
- 그 경우 데스크톱 앱이 진입점이 되고, cairn 백엔드가 로컬 데몬으로 도는 구조가 자연스러움.

## 후보 스택 (메모, 미결정)

- Tauri (Rust + 웹뷰) — 번들 크기 작고 시스템 친화적
- Electron — 생태계 풍부하지만 번들 무거움
- 네이티브 SwiftUI (macOS 한정이면) — 본인 환경이 macOS 고정이라 선택지로 가능

## 풀어야 할 것 (미리 던져 두는 질문)

- 데이터 소스: Notion을 계속 source-of-truth로 둘지, 로컬 SQLite/DuckDB에 한 번 더 인덱싱할지.
  - 후자면 **오프라인에서도 빠르게 검색·필터** 가능. cairn의 "로컬만, 클라우드 X" 원칙과도 결이 맞음.
- 일지/PR/커밋의 통합 타임라인 모델 (어떤 키로 묶을지: 날짜, 프로젝트, 태그?).
- 앱이 cairn 코어 기능(수집·요약·발행)도 트리거하게 할지, 그냥 뷰어로 둘지.
- 범용화 시 "코드 본문 외부 송신 금지" 원칙(→ ADR 0003)을 다른 사용자에게도 강제할 UX/기본값.

## 다음

- 실행 시점은 미정. 단계 1(수집 레이어) → 2(요약·발행) → 3(스케줄링) 까지 끝내고, **자기 일지가 충분히 쌓여서 "보는 도구"가 실제로 필요해질 때** 다시 꺼내 본다.
- 그 시점에 plan(`docs/plans/`)이나 ADR로 승격.
